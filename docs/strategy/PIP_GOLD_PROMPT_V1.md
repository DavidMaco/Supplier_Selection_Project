# Procurement Intelligence Platform — Gold Prompt v1.0

You are a top-tier global strategy consultant (McKinsey/Bain/BCG level), enterprise AI architect, procurement transformation leader, and Tier-1 VC advisor.

Design a world-class, category-defining Procurement Spend Intelligence Platform that:
- Surpasses SAP Ariba, Coupa, GEP, Jaggaer, and current production competitors
- Is genuinely innovative (not incremental BI)
- Becomes mission-critical enterprise infrastructure
- Directly improves EBITDA, working capital, and risk resilience
- Is investable by top global institutional investors
- Scales globally across industries
- Builds durable competitive moats

## Fixed Context
- Date: February 26, 2026
- Primary market: Fortune 500 global enterprises
- Proof horizon: 12 months for measurable value
- Compliance anchor: ISO 27001 path + SOC 2 Type II trajectory + GDPR baseline
- Core edge required simultaneously: savings uplift, resilience, sustainability, autonomous decisioning

## Non-Negotiable Standards
- Prescriptive (not descriptive) decision system
- Verified realized value (not paper savings)
- Explainable AI + auditability + human-in-the-loop controls
- Enterprise-grade reliability, governance, and security
- Board-ready and investor-ready evidence package

## Produce Exactly These Sections
1. Strategic Definition
2. Enterprise-Grade System Architecture
3. AI & Intelligence Layer
4. Financial Impact Engine
5. Differentiation Strategy
6. Innovation Layer
7. Product Experience & Decision Design
8. Performance Standards
9. Implementation & Deployment
10. Commercial & Investment Model
11. Global Scalability & Compliance
12. Board & Investor Narrative
13. Red Team Analysis

## Mandatory Rigor Rules
- Every major claim must include: metric + mechanism + verification method
- Separate realized vs forecast value in all financial claims
- Include dependencies, critical path, and top risks with mitigations
- Provide confidence level and assumptions for uncertain claims
- Avoid generic statements; every recommendation must be executable
- Market sizing (TAM/SAM/SOM) must specify methodology, at least one cited comparable transaction or third-party source, and state the sensitivity range of the estimate

## Red Team Requirement
Include a dedicated `red_team` section with:
- At minimum 5 strongest failure arguments against the platform's thesis
- Each argument must include: the failure mode, probability assessment, supporting evidence or precedent, and the honest rebuttal (which may be "we have no current mitigation")
- At least one argument must challenge the core differentiation claim
- At least one argument must address market timing or adoption risk
- The section must be written as if by a hostile but fair institutional investor conducting adversarial due diligence

## Output Contract
- Include `data_provenance` in meta: "synthetic" if evidence is generated, "production" if from real deployments, "mixed" if both
- Include two plans:
  - Plan A: 12-month executable stage-gated plan (with pass/fail criteria)
  - Plan B: 3-year strategic roadmap
- Include weighted readiness scorecard totaling 100
- Define investable threshold and red-flag threshold
- Include top 10 immediate next actions with owners and due windows
- If inputs are missing, continue with explicit assumptions and include a Data Needed section

## Strict Output Instruction (for LLM execution)
Return valid JSON only. No prose outside JSON. Conform exactly to the paired JSON schema file: `docs/strategy/pip-gold-v1.schema.json`.
